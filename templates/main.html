Important Concepts

Lesson 1: Basics of the Web and HTML

Overview of the Internet

Browser makes a request via the internet-the world's largest computer network, to the servers via HTTP. Servers respond with files that the browser displays. HTML is the main document type of the web. It stands for Hypertext Markup Language. It is made up of text content(what you see), markup(what it looks like) and references ( ex: images, videos and other documents).

Introduction to HTML

  • Tag: An HTML tag is contained within angled brackets. Most tags have an opening tag and a closing tag. Some tags are void and they do not require a closing tag. Break or Br is an example of a void tag.
  • Element: An HTML element refers to everything within a set of opening and closing tags.
  • Attribute: This is a property of an HTML element. Ex: image tag = <img src ="url"> , making links using the anchor tag = <a href"___">.
  • Inline vs Block : break and span are inline and p and div are block. Inline tag just ends a line whereas the block makes an invisible box around text which can have height and width. Moreover, span and div contain content and you can attach styles to them to adjust the way they behave.

Why are computers considered stupid?

Computers are considered stupid because programmers need to write exactly the way a computer understands or write with correct "syntax". This is good because it gives you the power and if you know how to talk to a computer you can tell it to do anything you want!

Tree Like Structure

  • HTML elements have more elements inside of them. There are boxes inside of boxes and so on surrounded by an opening tag and a closing tag.
  • When programmers talk about the DOM(Document Object Model) they are talking about the tree like structure of a page.
  • HTML is the language of the web and it has syntax and rules which are written in a document using HTML syntax and rules. Basic rules are tags.
  • Browser turns them in to elements that turn to a tree that uses DOM. It is a standard convention that interacts with elements in HTML.
  • HTML tag creates an element in the DOM that the browser uses to display the page. Element is created from the starting tag and the ending tag. Everything in between is content.
  • Parent-Child Relationship of HTML elements: Elements inside others have a parent-child relationship while elements with the same parent have a sibling relationship. Styling errors may appear when the nested elements don't line up correctly. For example from line 104 to 120 <div> is the top most element, <ul> is it's child.
  • CSS is used to style the elements displayed on the page. HTML tells the browser information on what style to use and where to get it.

Lesson 4: Introduction to Networks

Servers

Purpose of servers is to respond to HTTP requests. Web apps live on a server, speaks to HTTP and generates content that your browser requests. For example, when a user enters a url in the address bar the browser sends an HTTP GET reqest to the server. Server deciphers the HTTP request, runs the appropriate handler which either responds with static or dynamic content in the form of an HTML document. Static ones are prewritten. ex: Images, HTML. Dynamic are made on the fly by web applications. When a web application thats hosted on a server, gets a request it can respond with dynamically generated content. Example of an HTTP request : ____.

  • 100 Continue - the client should continue with its request.
  • 200 OK - the request has succeeded.
  • 402 payment required - the code is reserved for future use.
  • 400 Bad Request - the request could not be understood by the server due to malformed syntax and the client should not repeat the request without modifications.

Importance of Validating inputs

Validating inputs will help your application to prevent vulnerabilities exploited by malicious users(Ex: instead of a valid input someone can add HTML in our code and interfere).For example "> should not be allowed in the input. This can be fixed by escaping. Using dropdowns can limit what the user can actively enter. The other method is to verify what the users enter and complain if the data is bad. Validation can be done on the server side. Information is sent to the server and validated using a server-side language. In case the validation fails, the response is sent to the client and feedback is given. Client side validation can be done on the client using languages such as JavaScript which allows to validate input as it is being typed. This allows for a richer, more responsive validation. A drawback of this is that if the user turns of Javascript they can easily bypass the validation. So validation should be implemented on both the client side AND the server side. This will allow for the best user experience that is more secure with a fast response.
